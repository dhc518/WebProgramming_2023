액션 태그
	jsp에서 기본으로 제공하는 태그들의 입합으로 서버 또는 클라이언트에게 수행할 명령을 지시
	java코드를 사용하지 않아도 jsp웹페이지를 개발 가능
	<jsp:.../> 형태로 사용, 해당 태그는 반드시 /> 로 마무리


forward 태그
	현재 위치한 jsp웹 페이지 ㅇ서 다른 웹페이지로 이동할 때 사용
	<jsp:foirward page="파일명" />

include 태그
	현재 위치한 jsp웹페이지의 특정 영역에 외부 파일의 내용을 포함할 때 사용하는 태그
	flush 속성을 true로 할 경우 버퍼에 저장된 내용을 pop 및 출력, 헤더 내용도 같이 출력
	false(기본값)로 두고 사용하는 것이 좋음
	<jsp:include page="파일명" flush="false" />

param 태그
	현재 위치한 jsp웹페이지에서 다른 웹페이지로 정보를 전달할 때 사용하는 태그
	단독으로 사용불가, forward나 include 태긔의 내부에 선언하여 사용
	<jsp:foirward page="파일명">
		<jsp:param="파아미터명1" value="파라미터값1">
		<jsp:param="파아미터명2" value="파라미터값2">
	</jsp:foirward>

	request.getParameter("파라미터명")//파라미터 받기

java Beans 액션태그
	jsp웹페이지에서 java 소스 코드만 따로 뽑아 별도로 작성하여 페이지가 복잡해지는 것을 방지
	useBean 태그는 자바빈즈를 jsp웹페이지에서 사용하기 위해 java클래스를 선언하고 초기화하는 태그
	<jsp:useBean id="객체명" scope="page|request|session|application" class="클래스명" type="타입명"/>

useBean 태그 속성
	id속성
		꺼낸 객체의 참조 변수명으로 사용
		getAttribute()로 값을 꺼낼 때 사용하는 이름
		객체를 생성할 경우 보관소에 저장하는 key값의 이름으로 사용
	spcope속성
		page = JspContext(기본값, 현재 페이지에서만 사용가능)\
		request = ServletRequest
		session= HttpSession
		application = SevletContext
	class속성
		사용할 클래스 이름을 저정할 때 선언
		new 연산자를 사용하기 때문에 인테페이스는 올 수 없으며 반드시 패키지 이름을 포함
		scope에서 지정한 보관소에서 객체를 찾지 못하였을 경우 class의 값을 사용하여 인스턴스 생성
		생성된 객체는 보관소에 자동으로 저장, class속성을 선언하지 않을 경우 객체 생성 불가
	type속성
		참조변수에서 사용할 타입을 지정할 때 선언
		반드시 패키지 이름을 포함해야 하며 type속성을 지정하지 않으면 class속성의 값이 사용
